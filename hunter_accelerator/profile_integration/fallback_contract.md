# Fallback contract

The unmodified Hunter All profile is always authoritative.

- **COMPLETE:** use the deterministic baseline, retain every mandatory matcher and logic target, inspect novel surfaces, and continue all original Hunter All investigation and later phases.
- **PARTIAL:** use supported artifacts only. Run the original matcher process for every unresolved class, coverage gap, skipped security-relevant file and unsupported construct. Do not infer N/A or safety from the partial output.
- **FAILED:** disregard accelerator conclusions and run the original matcher process in full.
- **Contradictory evidence:** repository evidence wins. Re-run or bypass the accelerator and flag the contradiction as a coverage issue.
- **Downstream requirements:** source–sink–path proof, mitigations, reachability, verbatim evidence, N/A re-audit, logic decisions, triage, validation and reporting remain exclusively governed by Hunter All.

The accelerator never authorizes a final finding, dismissal, N/A re-audit result, CVSS score, remediation, target write, build, network request, or model call.
